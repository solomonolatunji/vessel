Codedock includes built-in tools to monitor server health, automatically clean up unused resources, and update itself without any manual intervention.

## Application Maintenance Mode

Codedock allows you to instantly place any running Application Service into **Maintenance Mode**.

- **What it does**: When enabled, the underlying container is gracefully stopped, and an `nginx:alpine` container serving a static "Under Maintenance" HTML page takes its place using the same routing rules and domain names.
- **When to use it**: This is extremely useful when you need to perform manual, time-consuming database migrations, heavy application updates, or emergency patching without throwing 502 Bad Gateway errors to your users.
- **How to toggle**: Navigate to the specific Service's **Configuration** tab in the Dashboard, find the Maintenance Mode card, and toggle the switch. All web traffic will be instantly routed to the maintenance page.

## System Maintenance

Codedock proactively tracks disk and resource usage to prevent your server from running out of space.

### What Codedock Measures

- Root filesystem disk usage.
- Docker storage and reclaimable data.
- Build artifact and cache size.
- Database backup storage size.

### Automated Cleanup (Garbage Collection)

You can configure Codedock to run a background cleanup task via the **Docker Cleanup Cron** setting in the dashboard. This task safely removes:

- Stopped, orphaned Docker containers.
- Unused, dangling Docker images.
- Unused Docker build caches.
- Old Codedock build artifacts.

Codedock explicitly *avoids* pruning Docker volumes automatically, ensuring your database data is never accidentally deleted even if a container is temporarily stopped.

### Disk Usage Alerts

Configure the **Disk Usage Threshold** (e.g., `80%`) and a cron schedule for checks. If your server crosses this threshold, Codedock will alert you via your configured notification channels (Discord, Slack, Email) so you can intervene before a catastrophic disk-full error occurs.

## System Updates

Codedock can keep itself updated to the latest version automatically.

### Auto-Updates

Because Codedock runs as a set of Docker containers (`codedockd`, `codedock-ui`, `traefik`), updates are incredibly clean.

When **Auto Update Enabled** is turned on in System Settings, Codedock will:
<Steps>
1. Check for a new release based on the **Update Check Cron** schedule.
2. If a new version is detected, pull the latest Docker images.
3. Gracefully restart the `codedockd` container.
</Steps>

Your running applications and databases will *not* go offline during a Codedock update. Only the dashboard and control plane API will be momentarily unavailable while the container restarts.

### Manual Updates

If you prefer to update Codedock manually, you can turn off Auto Updates. When a new version is available, it will be highlighted in the dashboard.

You can trigger a manual update via the UI, or by running the following on your server:

```bash
codedockd update
```
