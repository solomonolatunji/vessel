Codedock provides project-based team collaboration. While the system operates with global user accounts, permissions and access are scoped to individual projects.

## Project Members

You can invite other users to collaborate on your Codedock projects. Members have access to all environments, services, and databases within that specific project.

### Inviting Members

<Steps>
1. Navigate to your project in the Codedock dashboard.
2. Go to **Settings → Members**.
3. Enter the email address of the user you want to invite.
4. Select their **Permission level**.
5. Click **Invite**.
</Steps>

The invited user will receive an email containing a secure invitation link.

### Permission Levels

When inviting a member, you can assign them one of the following roles:

- **Can Edit**: The member has full read and write access to the project. They can deploy services, change environment variables, provision databases, and run SQL queries. They cannot, however, delete the project itself.
- **Read Only** (Coming soon): The member can view metrics, logs, and configuration, but cannot trigger deployments or modify any settings.

### Managing Invitations

Pending invitations are listed in the Members settings. You can revoke a pending invitation at any time before the user accepts it.

Once a user accepts the invitation, they will be listed as an active team member.

## Removing Members

Project administrators can remove members at any time:

<Steps>
1. Go to **Settings → Members**.
2. Find the active member.
3. Click the action menu and select **Remove**.
</Steps>

The member will immediately lose access to the project's dashboard, logs, and services.
