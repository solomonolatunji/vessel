Codedock turns any bare-metal VPS into your own private Platform-as-a-Service in under 60 seconds.

## One-Line Install

<Tabs>
  <TabItem label="curl" icon="seti:shell">
    ```sh curl -fsSL https://get.codedock.run | sh ```
  </TabItem>
  <TabItem label="wget" icon="seti:shell">
    ```sh wget -qO- https://get.codedock.run | sh ```
  </TabItem>
</Tabs>

This installs the `codedockd` daemon, pulls the required Docker images, and starts the dashboard at `http://<your-ip>:8080`.

<Aside type="tip" title="Prerequisites">

- A Linux VPS (Ubuntu 22.04+, Debian 12+, or any modern distro with kernel 5.x+)
- Docker Engine 24+ (installed automatically if missing)
- A domain pointing to your server (recommended for automatic SSL)
- Ports 80 and 443 open (for Traefik reverse proxy)

</Aside>

## Post-Install

After the install script completes, you'll see:

```text
✅ Codedock vlatest installed successfully!
  📍 Dashboard:    http://203.0.113.42:8080
  🛠️  Server CLI:  codedockd --help
  💻 Remote CLI:   curl -fsSL https://get.codedock.run/cli | sh
```

### Create Your Account

You have two ways to create the admin account:

<Tabs>
  <TabItem label="Via Dashboard" icon="laptop">
  Open `http://<your-server-ip>:8080` in your browser. The first user to register becomes the **instance admin**.
  </TabItem>
  <TabItem label="Via Terminal" icon="terminal">
  Run the setup wizard directly from your server (no browser needed):
  ```sh
  codedockd setup
  ```
  </TabItem>
</Tabs>

### Server CLI (`codedockd`)

The install script places `codedockd` at `/usr/local/bin/codedockd`. This is a shell wrapper that runs commands **inside the Docker container** on your server:

```sh
codedockd status            # Show daemon health and running containers
codedockd logs -f           # Tail daemon logs
codedockd reset-password    # Reset admin password
codedockd config            # View server configuration
codedockd config site-name=MyCodedock  # Update a setting
codedockd backup            # Create a manual database backup
codedockd update            # Upgrade to the latest version
codedockd downgrade v0.1.0  # Downgrade to a specific version
```

### Remote CLI (`codedock`)

Install on your **local machine** to manage your server from anywhere:

```sh
curl -fsSL https://get.codedock.run/cli | sh
codedock login    # Connect to your server
codedock project list
codedock deploy <service-id>
```

### Configure a Domain (Recommended)

Without a domain, Codedock assigns a magic DNS subdomain (like `sslip.io` or `traefik.me`) to every service. For production:

<Steps>

1. Go to **Settings → Server Settings**.
2. Set your wildcard domain (e.g. `*.codedock.example.com`).
3. Point an `A` record or `CNAME` to your server.
4. Traefik provisions Let's Encrypt SSL certificates automatically.

</Steps>
